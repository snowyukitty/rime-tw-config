# tw_fix.txt — luna_pinyin_tw 字形補正對照表

`luna_pinyin` 基礎詞庫沿用多個舊字形／異體字，而台灣字形轉換設定 `t2tw.json`
只套用 `TWVariants`，部分舊字形（最明顯的是「爲」）不會被正規化成教育部正字。

本表透過第二道 OpenCC simplifier（`simplifier@tw_fix`，見 `luna_pinyin_tw.custom.yaml`）
在 `t2tw` 之後再做一次字形正規化，涵蓋基礎詞庫、自學詞庫與未來造詞的所有候選。

格式：每行 `原字<Tab>台灣正體`（OpenCC 文字字典，不可含註解，故說明集中於本檔）。

## 取捨原則
- 只收「純字形、無語義分裂、台灣正體為單一標準字」者。
- 已是台灣正字者不動：`卻`、`黃`、`牆`。
- 實務通用 / 有語義差 / 文言保留者排除：`污`(汙)、`慾`(欲)、`喫`(吃)、`棱`(稜)、`廼`。
- 若日後想增刪，直接編輯 `opencc/tw_fix.txt`，一行一組，重新部署即可。

## 根因與處置（重要）
真正源頭是 `luna_pinyin.dict.yaml`（rime/rime-pinyin 上游）仍用舊字形。

- **主修法（已採用，跨所有 schema 生效、已驗證）**：直接把共享詞庫
  `luna_pinyin.dict.yaml` 內這 15 組字就地改為正字，重新部署後 `luna_pinyin.table.bin`
  即不含舊字形。重跑腳本見 `rime-config/patch-luna-dict.ps1`（小狼毫升級覆蓋詞庫後要重跑）。
- **本 OpenCC tw_fix 層**：保留為「輸出層」防護網（僅作用於 luna_pinyin_tw），
  涵蓋詞庫被覆蓋、或自學詞庫殘留舊字形的情況。注意它**只在 luna_pinyin_tw 生效**，
  在 luna_pinyin_simp 不會套用——這也是為何單靠它在 simp schema 下看不到效果。
- **未來 PR**：上述字形對照即可作為向上游 rime/rime-pinyin 提交的依據。
