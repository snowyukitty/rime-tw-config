# 字形正規化 實作紀錄（2026-06-26）

針對「打詞組會吐舊字形『爲』（如 無能爲力、因爲）」的修正全紀錄。

## 1. 問題與根因
- 現象：`luna_pinyin_tw` / 繁體輸出時出現「爲」「裏」等舊字形，而非台灣正字「為」「裡」。
- 根因（實測確認）：
  1. 基礎詞庫 `luna_pinyin.dict.yaml`（rime/rime-pinyin 上游）沿用舊字形，
     單字「爲」+ 82 個詞條使用「爲」。
  2. `luna_pinyin_tw` 的 `simplifier` 用 `opencc_config: t2tw.json`，
     但 `t2tw` 只套 `TWVariants`，**並不包含「爲→為」**，故原樣輸出。
  3. ⚠️ 更正先前 README 的說法：「走 t2tw 就會得到為」**不正確**——
     t2tw 會處理眾/裡等，但**漏掉爲**。

## 2. 解法（三層，替換非刪除）
| 層 | 檔案 | 作用 | 範圍 |
|---|---|---|---|
| ① 基礎詞庫直改（主修法） | `rime-config/patch-luna-dict.ps1` | 把 15 組舊字形就地換成正字，重編 `luna_pinyin.table.bin` | 所有 schema |
| ② OpenCC 輸出層補正（防護網） | `rime-config/opencc/tw_fix.{txt,json}` + `luna_pinyin_tw.custom.yaml` | `t2tw` 後再掛一道 `simplifier@tw_fix` | 僅 luna_pinyin_tw |
| ③ 自學詞庫正規化 | 直接就地替換快照字形 | 保留詞頻、僅改錯字 | 個人詞庫 |

處理的 15 組字形對照見 [`字形對照表.md`](字形對照表.md) 第一類。

### 關鍵：替換，不是刪除
- 基礎詞庫：替換前後 **70841 行 / 962079 bytes 完全相同**，詞條一筆未減。
- 自學詞庫：~2.1 萬筆詞頻全保留；行數的微幅變動是「字形統一後同 key 合併」，非刪詞。

## 3. 其他調整
- **預設 schema 固定為繁體**：`schema_list` 僅保留 `luna_pinyin_tw`，
  `user.yaml` 的 `previously_selected_schema` 設為 `luna_pinyin_tw`（原為
  `luna_pinyin_simp`）。簡體輸出改由 `Ctrl+Shift+4` 切 `zh_simp`，
  不再切到獨立的 `luna_pinyin_simp` schema。
- **熱鍵相容修正（2026-06-27）**：同時綁定 `Control+Shift+4` 與
  `Control+Shift+dollar`，避免 Windows / Rime 實際送出的 key event 不同而失效。
- **`.gitattributes`**：`tw_fix.txt` 釘 LF（避免 CRLF 把 `為\r` 寫進對應值）；
  userdb 快照保持 CRLF。

## 4. 驗證
- 源詞庫：舊字形 = 0，新字形已寫入（為=83、著=419、裡=53）。
- 編譯後 `luna_pinyin.table.bin`（位元組級）：舊字形 = 0。
- 自學詞庫匯出快照：0 爲、~2.1 萬筆。
- 執行期 log：`tw_fix.json` 成功初始化、零錯誤
  → 確認 librime 的 OpenCC **支援 `type: text` 文字字典**。

## 5. 已知限制 / 維運
- 小狼毫**升級或重裝會覆蓋**共享詞庫 → 需重跑 `patch-luna-dict.ps1` 再部署。
  （故保留 ② OpenCC 層作為覆蓋後的防護網。）
- 操作 userdb 需先**停 WeaselServer 釋放 LevelDB 鎖**，`/sync` 才匯得出
  （見下「backup 隱患」）。
- 一簡多繁（后/裡/面…）屬語義分流，**不可**全域替換，靠詞庫詞組正確收錄。
